# Service Desk Database – Учет заявок на доработку внутренних сервисов

## Описание проекта

Данный проект представляет собой реляционную базу данных для централизованного учёта заявок на доработку внутренних сервисов. Разработан в рамках производственной практики в ООО КБ «Ренессанс Банк» (отдел развития внутренних сервисов).

База данных позволяет:
- регистрировать заявки с указанием заявителя, сервиса, приоритета и статуса;
- назначать ответственных разработчиков;
- вести историю комментариев и вложений;
- формировать отчёты по загрузке сотрудников и срокам выполнения.

---

## Технологии

- **СУБД:** PostgreSQL
- **Инструменты моделирования:** dbdiagram.io
- **Язык:** SQL (DDL, DML, DCL)
- **Система контроля версий:** Git

---

## Структура базы данных

Основные таблицы:
- `Users` – сотрудники (заявители, разработчики, руководители)
- `Services` – внутренние сервисы
- `Statuses` – справочник статусов заявок
- `Priorities` – справочник приоритетов
- `Tickets` – заявки (основная таблица)
- `Developers` – разработчики (связь с Users)
- `Assignments` – назначения заявок на разработчиков
- `Comments` – комментарии к заявкам
- `Attachments` – вложения (файлы)

Связи между таблицами и ER-диаграммы представлены в отчете и на изображении ERD.png.

## Установка и развертывание

1. Убедитесь, что PostgreSQL установлен и запущен.

2. Создайте пустую базу данных (используйте `template0` для обхода ошибок колляции):
```sql
CREATE DATABASE service_desk TEMPLATE template0;
```
3. Подключитесь к созданной базе данных.
4. Откройте и выполните скрипты создания таблиц (DBAndTablesCreate.sql).
5. При необходимости откройте и выполните скрипты создания ролей и представлений (Roles.sql, Views.sql).
